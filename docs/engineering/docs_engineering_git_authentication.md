# Git 认证通道与稳定性说明（工程版）

> 适用范围：本项目的所有开发、CI、本地开发与自动化脚本

------------------------------------------------------------------------

## 背景

在 Git
使用过程中，认证失败并不一定是"配置错误"，而常常是**认证通道发生变化**导致。为了保证开发流程的稳定性，本项目统一采用
**SSH 作为唯一写入通道**。

------------------------------------------------------------------------

## 为什么不推荐 HTTPS 作为写入通道

HTTPS 依赖以下动态因素：

-   VSCode AskPass / Socket
-   macOS Keychain
-   GitHub Token（PAT）
-   运行时环境变量

这些因素会因为：系统更新、VSCode 状态变化、token
失效等原因发生改变，导致不稳定。

------------------------------------------------------------------------

## 本项目的标准方案：SSH

### 优点

  维度                SSH
  ------------------- -------
  稳定性              ✅ 高
  可复现              ✅
  CI 友好             ✅
  无 token 过期问题   ✅
  安全性              ✅

------------------------------------------------------------------------

## 标准配置流程

### 1. 切换 remote 为 SSH

``` bash
git remote set-url origin git@github.com:jinsie-aitechfun-lab/jinsie-ai-agent-platform.git
```

### 2. 确认本机 SSH key

``` bash
ls ~/.ssh
```

若不存在，请生成：

``` bash
ssh-keygen -t ed25519
```

### 3. 测试连接

``` bash
ssh -T git@github.com
```

------------------------------------------------------------------------

## 结论

本项目不接受 HTTPS 写入通道作为稳定方案。\
所有开发者必须使用 SSH 进行 push。
