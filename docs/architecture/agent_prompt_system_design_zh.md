# Agent 系统提示词设计（工程化公开版）

## 一、设计背景

在 AI 应用工程中，**System Prompt** 承担的是“系统级约束与职责定义”的角色，
用于确保大模型输出**结构稳定、可解析、可复用**，而不是一次性对话结果。

本设计文档总结了一种**工程化、可维护的 Agent System Prompt 设计方法**，
适用于任务拆解、规划生成、结构化输出等 AI 应用场景。

> 本文档为 **公开安全版**，不包含任何学习阶段标记、时间信息或个人轨迹。

---

## 二、设计目标

该 System Prompt 的核心目标包括：

- 明确 Agent 的**角色职责边界**
- 约束输出格式，保证 **JSON 结构稳定**
- 降低下游程序解析与二次处理成本
- 为后续 RAG / Workflow / Agent 扩展预留接口

---

## 三、核心设计原则

### 1. 单一职责原则（SRP）

System Prompt 只负责：

- 定义「你是谁」
- 定义「你要输出什么结构」
- 定义「你不能做什么」

**不包含**：
- 示例任务
- 业务策略
- 用户引导逻辑

---

### 2. 强约束输出结构

通过明确的 JSON Schema：

- 防止模型自由发挥
- 保证输出字段可预测
- 支持前后端、Agent、Workflow 稳定接入

示例结构：

```json
{
  "task_summary": "string",
  "steps": [],
  "assumptions": [],
  "risks": []
}
```

---

### 3. 防幻觉与不确定性控制

设计中明确要求：

- 不确定信息必须显式标注为“未知”
- 禁止输出与任务无关内容
- 禁止输出额外解释性文本

---

## 四、工程演进视角

该 Prompt 结构可自然演进为：

- 多 Agent 角色系统（Planner / Executor / Reviewer）
- Workflow 节点输入输出标准
- CI / 自动评测中的结构校验单元

其价值不在于“写 Prompt”，而在于：

> **把 Prompt 当成工程接口，而不是对话技巧。**

---

## 五、适用场景

- AI 任务规划器（Task Planner）
- 任务 / 项目 / 执行计划生成
- 复杂任务的结构化拆解
- Agent 系统中的中枢调度 Prompt

---

## 六、总结

一个优秀的 System Prompt，应当：

- 像接口文档一样稳定
- 像协议一样可约束
- 像工程资产一样可维护

这正是 AI 应用工程化与“玩模型”的本质区别。
